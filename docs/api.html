JpaResultMapper

Utility class for mapping JPA Tuple results to typed objects. All methods are static. 将 JPA Tuple 结果映射为类型化对象的工具类，所有方法均为静态方法。

static <T> List<T> map(List<Tuple> tuples, Class<T> resultType)
Map a list of Tuples to a typed list.将 Tuple 列表映射为类型化列表。
static <T> T map(Tuple tuple, Class<T> resultType)
Map a single Tuple to a typed object. Dispatches to Map, Record, interface projection, DTO, or scalar conversion. Applies underscore-to-camelCase alias conversion.

DTO mapping strategy:
  • If the class has a no-arg constructor → BeanWrapper path (setter-based, existing behavior).
  • Otherwise → Constructor-based mapping: selects the constructor with the most parameters, resolves parameter names via @ConstructorProperties or -parameters reflection, matches Tuple aliases (with underscore-to-camelCase), applies type conversion, and uses primitive default values for null primitives.
This enables mapping to Kotlin data classes and immutable Java DTOs without a no-arg constructor. 将单个 Tuple 映射为类型化对象。分发到 Map、Record、接口投影、DTO 或标量转换。自动应用下划线转驼峰别名转换。

DTO 映射策略：
  • 有无参构造器 → BeanWrapper 路径（基于 setter，现有行为）。
  • 无无参构造器 → 构造器推断映射：选择参数最多的构造器，通过 @ConstructorProperties 或 -parameters 反射解析参数名，匹配 Tuple 别名（含下划线转驼峰），自动类型转换，基本类型 null 安全。
支持 Kotlin data class 和无无参构造器的不可变 Java DTO 映射。

ParameterBinder

Utility class for binding named parameters to JPA queries. All methods are static. 将命名参数绑定到 JPA 查询的工具类，所有方法均为静态方法。

static void bind(Query query, Map<String, Object> params)
Bind Map parameters to a JPA Query. Only binds parameters that exist in the query — extra keys are safely ignored.将 Map 参数绑定到 JPA Query。仅绑定查询中存在的参数，多余的键会被安全忽略。
static void bind(Query query, Object paramObj)
Bind POJO fields as named parameters. If paramObj is a Map, delegates to the Map overload.将 POJO 字段作为命名参数绑定。如果 paramObj 是 Map，则委托给 Map 重载。
static Map<String, Object> toMap(Object obj)
Convert a POJO to Map using cached reflection. Traverses superclass hierarchy, skips static/synthetic fields.使用缓存反射将 POJO 转换为 Map。遍历父类层级，跳过 static/synthetic 字段。

BoosterSqlRewriter

Orchestrates SQL rewriting. Collects null/blank/empty parameters, removes dependent conditions via JSqlParser AST, and performs safety checks. 编排 SQL 改写流程。收集 null/空白/空集合参数，通过 JSqlParser AST 移除依赖条件，并执行安全检查。

static SqlRewriteResult rewrite(String sql, Map<String, Object> params)
Rewrite SQL by removing conditions for null/blank/empty parameters. Returns rewritten SQL and filtered parameters.通过移除 null/空白/空集合参数的条件来改写 SQL。返回改写后的 SQL 和过滤后的参数。
static Set<String> collectNullParams(Map<String, Object> params)
Collect parameter names whose values are null, blank, or empty collections.收集值为 null、空白或空集合的参数名。
static Map<String, Object> filterParams(Map<String, Object> params, Set<String> nullParams, String rewrittenSql)
Filter out null parameters no longer referenced in the rewritten SQL. Parameters still referenced (e.g. in :name IS NULL patterns) are retained with their value normalized to null. 过滤掉重写后 SQL 中不再引用的空参数。仍被引用的参数（如 :name IS NULL 模式）保留，值归一化为 null。
static Set<String> extractParamNames(String sql)
Extract all named parameter names from a SQL string. Supports underscore-prefixed names (e.g. :_status). Excludes PostgreSQL double-colon casts (::type). 从 SQL 字符串中提取所有命名参数名称。支持下划线开头的参数名（如 :_status），排除 PostgreSQL 双冒号类型转换（::type）。
record SqlRewriteResult(String sql, Map<String, Object> params)
Result record containing rewritten SQL and filtered parameters.包含改写后 SQL 和过滤后参数的结果记录。

BoosterCache

Cache interface for storing rewritten SQL. Default implementation uses Caffeine. 存储改写后 SQL 的缓存接口，默认实现使用 Caffeine。

String get(Object key)
Get cached SQL by key. Returns null if not found.根据 key 获取缓存的 SQL，未找到返回 null。
void put(Object key, String value)
Store SQL in cache.将 SQL 存入缓存。
void clear()
Clear all cached entries.清除所有缓存条目。
