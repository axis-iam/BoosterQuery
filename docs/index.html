BoosterQuery

A lightweight Spring Data JPA library for native SQL execution, automatic rewriting, and result mapping. 轻量级 Spring Data JPA 原生 SQL 增强库，提供智能改写与自动结果映射。

Core Features核心特性

⚡

Native SQL Execution

Execute native SQL with unified API. Supports paged queries, list queries, single-object queries, count, and DML operations. 统一 API 执行原生 SQL，支持分页查询、列表查询、单条查询、计数和 DML 操作。

✨

SQL Rewriting

Automatically removes WHERE/HAVING/JOIN conditions when parameters are null, blank, or empty. AST-based rewriting via JSqlParser. 参数为 null、空白或空集合时自动移除 WHERE/HAVING/JOIN 条件，基于 JSqlParser AST 改写。

🔄

Result Mapping

Map Tuple results to Entity, DTO, Record, Map, or primitive types. Automatic underscore-to-camelCase alias conversion. 将 Tuple 结果映射为 Entity、DTO、Record、Map 或基础类型，自动下划线转驼峰。

🛡

Auto-Limit Protection

Prevents large result sets by automatically appending LIMIT to queries. Configurable default limit (10,000 rows). 自动为查询追加 LIMIT 防止大结果集，默认限制 10,000 行，可配置。

⚡

Caffeine Caching

Cache rewritten SQL, count SQL, and sort SQL with Caffeine. Configurable size and TTL. 使用 Caffeine 缓存改写后的 SQL、计数 SQL 和排序 SQL，可配置大小和过期时间。

📄

@BoosterQuery Annotation

Declarative SQL queries on repository methods. Supports per-method rewrite/limit overrides and custom result types. 在 Repository 方法上声明式定义 SQL 查询，支持逐方法覆盖改写/限制配置和自定义返回类型。

Quick Start快速开始

Get up and running in 3 steps 3 步快速上手

1

Add Dependency添加依赖

implementation("com.chaosguide:booster-query:1.0.0")
<dependency>
    <groupId>com.chaosguide</groupId>
    <artifactId>booster-query</artifactId>
    <version>1.0.0</version>
</dependency>
2

Enable BoosterQuery启用 BoosterQuery

@SpringBootApplication
@EnableJpaRepositories(repositoryFactoryBeanClass = BoosterQueryRepositoryFactoryBean.class)
public class MyApplication { }
3

Use Enhanced Queries使用增强查询

public interface UserRepository extends BoosterQueryRepository<User, Long> { }

// Null parameters are automatically removed from WHERE clause
String sql = "SELECT * FROM t_user WHERE name = :name AND age > :age";
Map<String, Object> params = new HashMap<>();
params.put("name", "Alice");
params.put("age", null);  // null → condition removed
List<User> users = userRepo.boosterQueryList(sql, params);
// → SQL becomes: SELECT * FROM t_user WHERE name = :name
